Agent-Native Web (AWI)

A proposal for making websites reliably operable by browser agents.

What is AWI?

Websites today are built for humans and partially for assistive technologies via ARIA, but not for software agents. LLM agents can interact with websites, but they usually do it in brittle ways — guessing CSS selectors, relying on visual position, and breaking when UI labels or layouts change.

AWI is a proposal for a new semantic layer on the web, similar in spirit to ARIA, but focused on agent interaction. The idea: the UI remains the source of truth for humans, while the DOM exposes agent-readable semantics. A typed manifest provides stable capabilities. Tooling can auto-generate SDKs, CLIs, and browser automation wrappers.

This is not "replace the website with tools." The proposal is to make the website itself agent-readable. Agents would interact with the real UI through named actions and fields — never CSS selectors.

Proposed Architecture (4 Layers)

LayerNameDescription
1 UI Layer Human-facing website or web app. Unchanged — the source of truth for users.
2 Agent Semantics Layer DOM annotations using data-agent-* attributes that describe intent, fields, action types, risk, and scopes.
3 Agent Manifest Layer Typed JSON schemas at /.well-known/agent-manifest.json for actions, inputs, outputs, errors, scopes, and versioning.
4 Tooling Layer Generated from manifest and annotations: site SDKs (TypeScript, Python), site CLIs, MCP server wrappers, browser runtime helpers, and conformance tests.

Terminology

TermDefinition
Agent Software system (LLM or otherwise) interacting with a website.
Agent Semantics Machine-readable metadata describing UI intent.
Agent Action A named operation represented in UI or manifest (e.g. invoice.create).
Agent Field Logical input/output field name (e.g. customer_email).
Risk Level Metadata indicating action sensitivity: none, low, or high.
Confirmation Whether explicit confirmation is required before execution: never, optional, or required.
Manifest Typed machine-readable capability description at /.well-known/agent-manifest.json.
Generator Tool that creates SDK/CLI/MCP wrappers from semantics and manifest.
Agent Runtime Browser automation layer that executes UI interactions using semantics.
